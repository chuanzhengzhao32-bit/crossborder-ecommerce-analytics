# 跨境电商经营分析自动化工具原型

中文 | [English](README.md)

这是一个面向招聘方展示的跨境电商经营分析工具原型。它不是单纯的静态分析报告，而是把平台后台导出数据或未来 API 数据接入后，自动完成指标计算、利润分析、广告诊断、库存预警和经营复盘导出的解决方案雏形。

项目重点展示：我不仅能做一次性分析，也能把分析流程沉淀成业务团队可复用的工具。

## 方案逻辑

```text
平台后台 / CSV 导出 / 未来 API
  -> 标准化订单、成本、广告、库存表
  -> 自动校验字段和数据质量
  -> KPI 与利润计算引擎
  -> 广告和库存异常诊断
  -> Streamlit 经营分析页面
  -> 一键导出月度经营复盘
```

当前 MVP 支持样例数据和 CSV 上传。仓库里同时预留了 Shopify、Amazon、ERP、WMS、广告平台等 API 接入位置，用于说明真实业务中如何扩展。

> 数据边界：订单交易数据来自 UCI Online Retail II 真实公开数据；成本、广告和库存字段是为了作品集展示而构建的模拟经营数据，不代表任何真实公司的经营结果。

## 与岗位需求的对应

| 业务需求 | 工具能力 |
|---|---|
| 周报/月报经营分析 | 自动生成 KPI 总览和经营复盘 |
| 销售、毛利、费用、贡献利润 | 利润引擎覆盖收入、成本、平台费、支付费、物流费、广告和贡献利润 |
| 市场、渠道、品类、SKU 分析 | 输出分市场、分渠道、分品类和 SKU 维度的利润表现 |
| 广告费用利润分析 | 自动识别低 ROAS 广告组合并给出预算控制方向 |
| 库存周转、缺货、滞销分析 | 自动识别缺货风险 SKU 和滞销 SKU |
| 多源数据整合 | 支持 CSV 接入，并预留 API connector 结构 |
| BI / 数据建模能力 | 提供 SQL、数据字典、指标字典、模型说明和 DAX 指标 |

## 工具原型包含什么

Streamlit 页面支持：

- 使用本地完整数据、GitHub 样例数据或上传 CSV；
- 自动检查关键字段是否齐全；
- 展示经营 KPI 卡片；
- 展示月度收入、毛利、贡献利润趋势；
- 展示品类利润和市场利润；
- 展示低 ROAS 广告组合；
- 展示缺货和滞销 SKU；
- 下载 Markdown 月度经营复盘。

## 经营报表示例

最新完整分析月为 2011-11，对比 2011-10。

| 月份 | 有效订单 | 销售收入 | 毛利率 | 广告花费 | 贡献利润 | 贡献利润率 |
|---|---:|---:|---:|---:|---:|---:|
| 2011-10 | 2,040 | £1.15M | 40.0% | £26.1k | £225.1k | 19.5% |
| 2011-11 | 2,769 | £1.51M | 24.4% | £36.9k | £88.4k | 5.9% |

核心结论：

- 销售收入增长，但贡献利润下降，说明增长质量变差；
- 高销售额品类不一定高利润，需要结合成本和费用看贡献利润；
- 广告预算不能只看 ROAS，还要结合贡献利润和库存状态；
- 库存管理要区分“快断货 SKU”和“滞销占资 SKU”。

## 面试时可以这样介绍

我做了一个跨境电商经营分析工具原型。它可以把平台后台导出的订单、广告、库存和成本数据标准化，然后自动输出销售、毛利、贡献利润、ROAS、库存周转、缺货和滞销分析，并生成月度经营复盘。当前版本用 CSV 和样例数据演示，代码结构中预留了 Shopify、Amazon、ERP、WMS 和广告平台 API 的接入位置，展示的是把分析流程产品化和自动化的能力。
